# 第一部分 🏆 顶级 deck 共因

> 判据口径【出处:ai-skills/README】【原文】：「**结构对了才有效，不是因为模型特别**」——顶级 deck ≠ 漂亮，= 把咨询界成熟结构（金字塔/MECE/Day-1 假设）执行到位。

## 共因 1：每页标题是「论断」不是「主题」（claim-shaped title）
**机制**：标题陈述发现而非话题。好：「市场涨 40% 但收入跌 5%」；坏：「市场分析」。**这是两个明珠里出现频率最高、最一致的单一原则**（storyline / charts / critic / deck-pipeline / top-down-memo **至少 5 处反复编码**）。【出处:ai-skills 多处】【原文】
**为什么**：每个标题 = 一个 Lead，这页去证明它；**只读全 deck 标题就能复述出推荐**（金字塔在 slide 尺度的兑现）。
**deck 例**（✅ MVP 拆 3 份真实成品印证·可信值↑）：MTA 麦肯锡 p10「08/09 危机 ridership 呈 U 型…长路径恢复」(结论句·⭐⭐⭐⭐⭐) ↔ 某品牌 p2「品牌梳理」(主题词·营销 deck 容忍)。
**🌟 真实样本细化「action title 谱系」**：某品牌(主题词标题) ＜ 某品牌(功能描述) ＜ MTA(结论句)——**分品类**：咨询级**必须**结论句、投资/品牌 deck 容忍主题词+加粗关键句。〔3 份真实成品背书·已迈出从「强候选」升「铁律」第一步·详 [真实样本/_MVP拆解验证](../../research_lib/真实样本/_MVP拆解验证.md)〕

## 共因 2：每页都有 so-what + 综合≠摘要（≤3 条洞察）
**机制**【出处:ai-skills/synthesis】【原文】：唯一检验=「输出是否改变读者将如何行动？否=你写的是摘要，重写」；硬约束「**3 条洞察，绝不更多**」「每条以一句 so-what 收尾：填空『因为这为真，读者应当 ___』，填不出=洞察不够锐」「综合是减法，扔掉 80% 输入」。
**为什么**：deck 不是信息陈列，是「3 条带证据+蕴含的论断」。

## 共因 3：金字塔结论先行 · stop-anywhere
**机制**【出处:ai-skills/top-down-memo】【原文】：读者停在第一页就该知道你怎么想、要什么；停在任一层都带走正确要点。
**为什么**：高 stakes 受众是忙碌决策者，要先听结论。

## 共因 4：整个 deck = SCR 三段式
**机制**【出处:ai-skills/deck-pipeline + examples】【原文】：Situation（建共识）→ Complication（制造张力）→ Resolution（数据+结论）；实例 8 页显式标 S(2)/C(2)/R(4)。
**为什么**：读者能一眼定位「现在在故事哪」。

## 共因 5：末页是「决策强制」(decision-forcing)
**机制**【出处:ai-skills/decision-memo-builder】【原文】：末行是**带 owner + 日期的 yes/no ask**，绝不「需进一步分析」。
**为什么**：高 stakes deck 的目的是推动一个决策，不是汇报。

## 共因 6：先逻辑后排版（议题树→storyline→图→总装）
**机制**【出处:ai-skills/README + critic】【原文】：生产顺序 = 逻辑(树)→叙事(线)→可视化(图)→总装+自我批判；「**先跑 critic 再美化排版**」。
**为什么**：逻辑没立住前碰排版 = 给坏结构化妆。

## 共因 7：选图先选故事 · 一图一高亮 · 每图带 source
**机制**【出处:ai-skills/mckinsey-charts】【原文】：单数→bar_callout / 构成迁移→stacked_over_time / 桥接→waterfall；**一个高亮色其余全灰**（「麦肯锡 deck 不彩虹」）；少墨水多信号；**底部 source 永远要有**。
**为什么**：图表服务一个论断，不是装饰。

## 共因 8：图文互补（内容维度到顶的关卡）
**机制**【出处:PPTEval ppteval_content.txt】【原文】：内容评分纯文字上限 3 分；有图(过3)→图贴题(过4)→**图文互补**(到5)。
**为什么**：视觉辅助是内容传达的一部分，非点缀。

## 共因 9：开场/收尾的背景信息完整（连贯维度到顶的关卡）
**机制**【出处:PPTEval ppteval_coherence.txt】【原文】：连贯「逻辑通」只值 3 分；有基础背景(谁讲/何时/机构)(4)→**背景齐全+致谢结论**(5)。
**为什么**：拉开差距的是叙事感与元信息，不只是逻辑通顺。

## 📋 速查表

**做 deck 前（焊共因）**：□ 每页标题是带数字论断 □ 全 deck 走 S/C/R □ 先搭议题树/storyline 再排版 □ ≤3 条核心洞察 □ 末页 owner+日期的 ask。

**做完每页（查雷区，→ [05 质量门](05_质量门与评估.md)）**：
- 【机检可判】□ 每个数字有 source □ 标题 ≤15 词 □ 每页有图（非纯文字）
- 【人审】□ 标题是论断非主题 □ 有 so-what □ 无逻辑跳跃 □ 有明确下一步 □ 推荐具体（带数额/时限）

> **升格路径**：拆真实高 stakes deck → 给每条共因/雷区补真实 deck 例（名+页+段位）→ 人审 → 记 [_修订日志](_修订日志.md) 可信值 → 升为铁律。
