## 一、页型分类：12 大类 ~70 版式（直接复用为「按页型分类」骨架）

【出处:references/layout-catalog.md】【原文】引擎自带 A~M 12 大类（实际约 70 个活版式，#14 已废弃）：

- **C 框架矩阵**：**2×2 矩阵** / 阶梯进化 / 流程箭头 / 维恩图 / 殿堂框架
- **D 对比评估**：左右对比 / 前后对比 / 优劣 pros-cons / 红绿灯状态
- **E 内容叙事**：**执行摘要** / 核心洞见 / 引言 / 双栏文本 / 四栏概览
- **F 时间流程**：时间轴 / 垂直步骤 / 循环 / 漏斗
- **G 团队专题**：团队介绍 / 案例 / 行动项 / 结束页
- **H 柱图**：分组柱 / 堆叠柱 / 水平柱
- **I 图文版式**：右图 / 左图 / 三图 / 图+四要点 / 全宽背景图 / 带图案例 / 背景图引言 / 目标插图

- **K 仪表盘**：KPI+图+洞见 / 表+图+factoid
- **L 视觉叙事**：利益相关方地图 / 决策树 / 饼图 / SWOT / 议程 / 价值链 / 堆叠面积 …
- **M 编辑式叙事**：表格+洞见面板 / 多柱面板图（最接近 Mekko 但≠Mekko）

## 二、高 stakes 难图实现（护城河 · 重点）

### 2.1 瀑布图 / 桥梁图 waterfall #49 —— ✅ 有，纯 python-pptx 矩形拼
【出处:engine.py L2050-2102 `def waterfall`】【归纳】不调任何图表库，**逐根画矩形 `add_rect`** + 维护 `running` 累计值：base 从 0 起；up 段颜色 ACCENT_GREEN、`running+=val`；down 段颜色 ACCENT_RED。**桥接虚线**：第 i 根画一条短水平线把上一根顶和这一根连起来（瀑布标志性「台阶连接线」）。数值标签正值显 `+{val}`。**算法核心 = 累计变量 + 矩形堆叠 + 桥接短线**。这是「原生可编辑高级图表」的范例。

### 2.2 Mekko / Marimekko —— ❌ 明珠没有 = 护城河空位
【出处:全仓 grep `mekko|marimekko` 零命中】最接近的 multi_bar_panel 是「等宽柱面板」，**不是 Mekko 的「宽度=份额、高度=占比」二维编码**。→ **我们要啃的差异化点。** 可借鉴明珠的「旋转箭头连两点」技巧（`math.atan2` 求角 + `shape.rotation` + XML adj 调形）做 Mekko 的变宽列。

### 2.3 Gantt 甘特图 —— ❌ 明珠没有 = 护城河空位
【出处:全仓 grep `gantt` 零命中】最接近的 timeline/value_chain 都是「里程碑点/水平流程」，**没有「任务条按起止时间定位 + 时间轴刻度」的甘特结构**。→ **另一个差异化点。**

### 2.4 2×2 / 竞争矩阵 —— ✅ 有
【出处:engine.py L503-543 matrix_2x2 #13】四象限色块 + 轴标签（Y 竖排左、X 横排底）。另有 bubble #53（L2250）真正的「散点+气泡大小」二维编码，可当「竞争格局图（份额=气泡大小）」。

### 2.5 风险热力矩阵 #54 —— ✅ 有（3×3 硬编码）
【出处:engine.py L2294-2336】双层 for 画 9 格背景 + 小色点，风险名按 (row,col) 落格。**注意写死 3×3，不支持 NxN。**

### 2.6 ★圆形图护城河技巧：BLOCK_ARC 原生弧
【出处:core.py L204-258 `add_block_arc` + README.md L202-212】【原文/归纳】用 PowerPoint 原生形状 `MSO_SHAPE.BLOCK_ARC`，**直改 XML 三个调整柄** adj1(起始角)/adj2(结束角，换算成 1/60000 度)/adj3(内径比 0-50000)。**为何是护城河**：v1.x 一个甜甜圈要堆 **2800 个矩形**（~5MB、2 分钟）；v2.0 改 BLOCK_ARC 后**一个圆只要 3-4 个形状**（<1 秒、0.5MB）。→ donut/pie/gauge 全用它。**「吃透 PPT 原生形状 + 直改 OOXML」是通用 agent / 大厂 skill 做不出那类难图的通法。**

### 2.7 其它已实现高 stakes 图
Harvey Ball 表（白矩形遮罩出 0/¼/½/¾/满五档）/ 帕累托（降序柱+累计%）/ 决策树 / 阶梯图（连续台阶轮廓）。【出处:core.py L273-302, engine.py L2154/L616-639】

## 三、设计系统（配色 / 字号 / 网格 / 哲学 · 可直接当版式硬规范）

【出处:constants.py + references/team/brand-guide.md】【原文】
- **配色**：主色 NAVY `#051C2C`（标题/章节/高亮唯一主色）+ 灰阶四级（正文#333 / 次要#666 / 分隔#CCC / 底面板#F2F2F2）+ 强调四色（蓝#006BA6/绿#007A53/橙#D46A00/红#C62828，**仅当一页有 3+ 并列项才准用**）。CYAN 明令废弃。
- **字号 8 级**（严格）：44 封面 / 28 章节·TOC / 24 副标题 / **22 内容页 action title** / 18 栏头 / 16 强调 / **14 正文(主字号)** / 9 脚注。
- **字体**：英文标题 Georgia、正文 Arial；**中文全 KaiTi 楷体**。
- **网格(16:9)**：画布 13.333"×7.5"；左右边距 0.8"；内容区上界 1.3"；source 基线 7.05"。
- **设计哲学 4 条**：极简（无阴影/3D/渐变）· 一致 · 层级（22→18→14→9）· 平面（纯实色无效果）。

## 四、内容密度硬规则 + 版式能力边界（→ 机检料）

【出处:references/framework/planning-guide.md L60-70】【原文】每内容页 ≥3 视觉块；内容区使用率 ≥50%；**Action Title 必须完整句（表洞见非标签）**；有数字必突出；每页有 source；相邻页不得同版式；two_column_text 全局 ≤1 张。

【出处:references/layout-matrix.yaml】【原文，char_budget 真相源】donut/pie **≤6 段**（超则合并 top5+其他）；four_column **≤4 列**（描述 ≤120 字）；grouped_bar **≤6 类×3 系列**；process_chevron **≤5 步**；timeline 末 milestone 标签 ≤6 字符。→ **这些是可直接落 `deck_rules` 的确定性硬门。**

## 五、生产护栏 13 条 + QA 9 类（明珠的 fail-closed 原型 → 喂 deck_rules）

【出处:README.md L248-260 + qa.py】【原文】**QA 9 类检查**：body_overflow / text_overflow / dead_whitespace / shape_overlap / font_issue / text_line_collision / peer_font_inconsistency / chart_legend_overflow / guard_rail。**护栏 13 条**含：禁用 connector（防文件损坏）/ 每 shape 去阴影3D / 圆形图必用 BLOCK_ARC / 中文必设 EA 字体 / 文字-分隔线碰撞检查 / 生成后过 QA 门。

## 六、对本项目的启示（护城河决策）
1. **页型分类法直接复用** A~M 12 类。
2. **BLOCK_ARC + OOXML 直改** 是真护城河技巧，可迁移到我们要做的难图。
3. **Mekko / Gantt 明珠没有** → 正是「咨询级 vs 通用 PPT」的差异化空位，本项目应重点啃（可借明珠的旋转箭头/矩形拼/原生形状思路自研）。
4. **char_budget + QA 9 类 + 护栏 13 条** → 直接转化为 `reinforce/deck_rules` 的确定性机检门。

> **待验证**：版式清单与阈值来自单一开源实现，需拆真实咨询 deck 交叉确认「高 stakes 段到底常用哪些页型、阈值是否适配」。Mekko/Gantt 实现方案待立项研发。
