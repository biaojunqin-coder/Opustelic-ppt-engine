## 一、母框架：金字塔原理（Minto · 一切的根）

【出处:issuetrees/Pyramid.md】【原文】结论放塔顶，次级论点按重要性降序往下支撑。议题树/SCQA/假设树**都源自** Barbara Minto 金字塔原理。【出处:ai-skills/top-down-memo】【原文】金字塔原理 = BLUF（军方）= Minto Pyramid（咨询），**同一回事**。

**Stop-anywhere 铁律（高 stakes deck 最硬的标尺）**
- 【出处:ai-skills/top-down-memo】【原文】检验法：「读者读完**第一段/第一页**能否就停下，并已知道你怎么想、你要什么？能=自上而下；要读到结尾才找到答案=你写的是个故事（不合格）。」
- 【出处:top-down-memo】【原文】「金字塔对幻灯片标题同样成立：**每个 slide 标题就是一个 Lead，这页去证明它。**」
- 【归纳】**标尺**：整个 deck 是一座金字塔——执行摘要页=塔顶 Lead，章节页=支撑论点，内容页=证据；读者停在任一层都能带走正确要点。

**自上而下六步配方**【出处:issuetrees/Pyramid.md】【原文】：①写主题框 → ②定读者要回答的问题 → ③写答案 → ④确立 Situation（第一句无争议、读者必点头）→ ⑤引出 Complication（读者「我知道啊，所以呢？」）→ ⑥复核：冲突是否立刻引出那个问题，否则改问题。

## 二、四大结构化框架（按「何时用」选型）

| 框架 | 一句话 | 何时用 | 出处 |
|---|---|---|---|
| **SCQA/SCQH** | 情景→冲突→问题→答案(或假设) | deck 开篇/执行摘要的叙事骨架 | [issuetrees]SCQA.md |
| **议题树** | 把总领问题 MECE 拆成子问题 | **还不知道什么重要时**、问题求解早期 | [ai-skills]issue-tree-builder |
| **假设树** | Day-1 先押答案、结构化到能快速证伪 | 已有方向、要砍工作量时 | [ai-skills]hypothesis-tree |
| **SCPR** | 现状→催化变化→清晰问题→带时限的建议 | 更宽的战略叙事/高管执行摘要 | [ai-skills]scpr-framework |

### 2.1 SCQA / SCQH 四部件精确判据【出处:issuetrees/SCQA.md】【原文】
- **Situation**：先说一件读者**已知**的事建立相关性。铁律：「**必须是事实陈述，不是『哪里不对』**。『公寓墙是白的』是情景；『我不喜欢白墙』不是。」
- **Complication**：「最近情况变了……」在故事里制造张力，触发问题。
- **Question**：「那我们该怎么办？」从冲突合乎逻辑地浮现。
- **Hypothesis/Answer**：「我们需要……」先概述（完成引言）再拆解展开。
- **纪律**：每部件原则上**只给一句话**——「这种简洁逼你把事情真正蒸馏到底。」

### 2.2 议题树【出处:ai-skills/issue-tree-builder】【原文】
建树五步：①从总领问题(L0)起 → ②分解到「**根本面**而非行动」(L1，瞄准 3 分支) → ③每分支 2–3 关键假设(L2) → ④每层校验 MECE → ⑤每个末端可用数据检验。**「3 的法则」**（3 主分支，3–5 可接受）；**总深度只 2 层**。何时停拆：【issuetrees/Trees.md】【原文】「看你现在能否基于事实**否证**某选项——能否证就该停」。

### 2.3 假设树（砍 ⅔ 工作量的利器）【出处:ai-skills/hypothesis-tree】【原文】
五条设计选择：①必须押 Day-1 答案，「『我不知道』不是假设。猜一个。大声地、快速地犯错。」②**3 个子假设，不能更多**（要 5 个=顶层没成形）③每个叶子=「一个能杀死该分支的检验」（「『多做点研究』不是检验。『拉 4 家竞品定价比』才是」）④显式标信心，**先打低信心分支** ⑤**证伪 > 证实**。【原文】「跳过假设树是战略工作耗时变 3 倍的头号原因。」

## 三、MECE 五种切法（章节/论点切分菜单）

【出处:issuetrees/Trees.md，转自 Crafting Cases】【原文】MECE = 互斥 + 穷尽。五种切法：

1. **代数结构**：利润=收入−成本；收入=价格×量。**保证 MECE**，用于短期数值问题，「凭一点数据就能砍掉半棵树」。
2. **流程结构**：开头-中段-结尾，只用于可预测流程（如制造）。
3. **概念框架**：3C、4P、波特五力、People-Process-Systems。适合定性/长期/无量化数据。
4. **细分**：按年龄/产品线/月份切。「不给根因，给问题大概在哪」，擅长揭示「**mix effect 混合效应**」（均值变了其实是各细分权重变了）。
5. **反义词**：供给vs需求、内vs外、短vs长。帮任何问题搭结构，用作另一结构的更深一层。

**三条自检**【原文】：①第一层切法是否最有洞察（**第一层决定其余一切**）？②有没有漏（google 几篇文章，把冒出的议题对照结构，放不进=不 MECE）？③**「把结构讲给你的狗听，若你听起来不像正常人在过结构，那它八成太复杂或太 buzzword」**。

## 四、Storyline：每行 = 一页标题

【出处:ai-skills/storyline-builder】【原文】「Storyline 是演示的脊柱——**storyline 里每一行，变成最终 deck 里的一个 slide 标题。**」四特征：①每行=一个行动导向的 slide 标题 ②逻辑流 problem→context→analysis→solution ③**标题是「论断」不是「主题」** ④**读者只看标题就该读懂整个故事**（= 水平逻辑唯一测试）。

**五步建线法**【原文】：①认场景类型选模板 → ②定制流（替换占位、增删页、保持递进）→ ③写行动标题（每行完整信息带数据）→ ④验流（逻辑无跳跃、导向明确下一步）→ ⑤建页（每条=一页，标题=storyline，正文支撑标题）。

**六条禁忌（反向质检尺）**【原文】：主题式标题 / 缺 so-what / 逻辑跳跃 / 太碎（50 页讲 15 页的事）/ 没结尾 / 含糊用语（「表现良好」而非「收入增 25%」）。

## 五、三套「按场景」故事线模板（范本骨架）

【出处:ai-skills/storyline-builder】【原文，填空式模板】——这是「范本清单按场景」最直接的料：

**模板 A · 市场战略 / Pitch Deck（12 行）**：市场规模($XXB,CAGR)→ 我们的细分 → Top3 竞品增速 → 我们的营收/排位 → 产品解决什么用例/客群 → Top10 客户构成 → 定价/ACV → 产品差异化 → 关键竞争优势(3 条)→ 三个增长机会 → 聚焦哪个 → 18 月路线图。

**模板 B · 内部问题求解（议题树式，13 行）**：问题陈述(现状vs目标)→ 三因素驱动 → 因素1/2/3 各贡献$XXM（**三页平行结构**）→ 根因洞察 → 三个解法 → 解法1/2/3（影响/投入/周期）→ 按影响/投入选定 → 实施路线图 → 成功指标。

**模板 C · 项目路线图（6 行）**：项目目标(可量化)→ 四阶段(Discovery→Design→Build→Launch)→ 各阶段周次/负责人/里程碑 → 团队构成+缺口 → 成功标准 → ROI 衡量。

> 三种通用叙事弧【原文】：Problem-to-Solution / Market-to-Strategy / Analysis-to-Action。

## 七、deck = SCR 三段式 + 端到端总装线

**标尺①｜SCR 三段式**【出处:ai-skills/deck-pipeline + examples/02-storyline.md】【原文】：deck 应能被一眼切成 Situation(建共识)→ Complication(制造张力)→ Resolution(数据+结论)。实例把 8 页显式标成 S(2页)/C(2页)/R(4页)。

**标尺②｜端到端生产 SOP**【出处:ai-skills/README.md「How the skills compose」】【原文，真实四步链】：
```
议题树（拆 3 个 MECE 分支+可检验假设）
```

## 八、叙事 → 图表的硬映射（选图先选故事）

【出处:ai-skills/mckinsey-charts】【原文】三种主力图 = 三种故事：
- **`bar_callout`**：当「一个数字就是故事」（TAM、市场规模、单一高亮数）。
- **`stacked_bar_over_time`**：当讲「随时间的构成/mix 变化」。
- **`waterfall`**：当「桥接两个数字」（收入桥/成本桥，start→adds→subtracts→end）。
- 【原文】何时**不**用：分布→直方图；散点/象限→另一种；>5 系列堆叠→拆 small multiples。

> 实例 chart slot【原文】：「Slide 3→bar+callout；Slide 5→waterfall(年度成本桥)；Slide 6→stacked column over time」——**叙事→图表类型是被明确映射的，不是临场决定。**

## 九、节奏 / 情绪起伏 / 演讲 vs 阅读 → 见 07 双模式

storyline 立住「论点 + 一组证据 + framing」后，**呈现层**还有一个总开关：deck 是被讲（演讲版）还是被读（阅读版）。这决定节奏曲线、情绪起伏、留白密度——详见 [07 演讲版vs阅读版](07_演讲版vs阅读版.md)（**真样本背书**：已拆 某品牌/Lulu 两份天花板）。
- **演讲版**有情绪起伏快慢节奏：封面→转场→情绪/slogan→数据 callout→高潮→落幕（呼应开篇）；
- **阅读版**匀速、每页带导读副标题、信息自洽。
- 用户洞察：**每页 = 一个观点 = 演讲者的汇总**；顶级 deck 有情绪起伏、有快慢节奏。
