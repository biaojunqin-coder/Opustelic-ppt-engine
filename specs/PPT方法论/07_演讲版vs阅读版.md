## 〇 总开关：先判交付形态（策略层阶段 1 必问·D19 FR3 起三选一）

动手搭 storyline 前，先确认：**这份 deck 主要是现场被讲，还是离线被读——还是先被客户自读、之后再由我们现场讲**（第三态「预读讲解版」·D19 扩展，定义见下方「四」节）？

双模式信号表（历史主干·场景明确命中"只讲"或"只读"时按此选）：

| 信号 | → 演讲版（讲） | → 阅读版（读） |
|---|---|---|
| 场景 | 路演 / 提案 / 发布会，有人在场讲 | 邮件传阅 / 评审 / 留底，无人讲 |
| 受众读法 | 跟着演讲者走 | 自己翻、自己读 |
| 成败关键 | 情绪、记忆点、说服现场 | 信息自洽、逻辑闭环、可追溯 |
| 选错的后果 | 信息太满 → 演讲者被 PPT 抢话 | 信息太空 → 读者看不懂 |

⚠️ 很多项目要**两版**：现场演讲版 + 留底阅读版。**同一 storyline（论点+证据+framing 不变），两套呈现。**

---

## 一、两套语言总表

| 维度 | 演讲版 | 阅读版 |
|---|---|---|
| 每页信息 | 一个焦点，极简 | 标题 + **导读副标题** + 内容，自洽 |
| 留白 | 大量 | 充分但密度高 |
| 配色 | 黑/白/橙/彩交替**造情绪分节奏** | 统一企业色，靠秩序 |
| 情绪页 | 必有（态度 / slogan / 高潮） | 几乎无，理性克制 |
| 数据 | 大 callout | 表格 / 线框 + 标注 |
| 转场 | Part X 纯标题页（节奏骨架） | 章节分隔，平稳 |
| 节奏 | 起伏（洞察→数据→高潮→落幕） | 匀速、模块化 |
| 收尾 | 情绪落幕，呼应开篇 | SOW 工作范围三栏 |
| 靠什么 | 演讲者 + 情绪 + 视觉 | 结构 + 一致性 + 自洽 |

---

## 二、演讲版生产规范

**节奏曲线（必须有起伏，不能匀速）**：
封面(极简) → 视觉化目录 → [**转场页** → 洞察情绪页 → 数据 callout → 案例] × N 章 → **情绪高潮页** → **落幕页**(呼应开篇)

**硬规则**：
- 每页正文 ≤ 极简阈值（信息靠演讲者口头补）；
- 每章 ≥1 张转场页（纯标题做节奏骨架）；
- 全程 ≥1 情绪/slogan 页 + ≥1 高潮页（否则"匀速、没高潮"）；
- 首尾呼应（落幕页回扣开篇）；
- 数据用大 callout，禁堆表格；
- 配色分节奏（黑=情绪 / 白=内容 / 橙=转场 / 彩=高潮）。

---

## 三、阅读版生产规范

**统一模板（每页三段式）**：标题(论点) + **导读副标题(这页在方案里干嘛)** + 内容

**硬规则**：
- **每页必有导读副标题**（标题下一行说明本页作用）——阅读版能脱离演讲者自读的关键，缺 = 不合格；
- 三段式模板统一；
- 每页信息自洽（脱离演讲者可读懂）；
- 侧边竖标签做章节导航；
- 收尾给可执行落点（SOW / next step）。

## 四、第三态「预读讲解版」（2026-07-02 扩展·D19 FR3·用户真实场景背书）

> **可信度定级：用户真实场景背书**——第二轮实测（059 IMC pitch·D19）用户提出的真实交付形态：*"方案先发给客户自己看，客户看完之后再开会由我们讲一遍。"* 两个旧态在这个场景下都不对：纯阅读版没为"后面那场讲"留讲述空间，纯演讲版发出去客户自己读不懂。**尚无真样本拆解背书**（双模式有 某品牌/Lulu 两份），待后续实测真实 deck 巩固升格。

**真实场景**：pitch / 提案的常见流程——deck 先行（发给客户预读）→ 客户带着已读印象进会 → 我们现场再讲一遍（强调 + 答疑）。**同一份 deck，先被"读"、再被"讲"**，两个阶段都要撑住。

**三特征（生产规范）**：

| 维度 | 取法 | 为什么 |
|---|---|---|
| 密度 | **取中**：比阅读版略疏、比演讲版满 | 自读要自洽（不能空到看不懂），现场讲要留讲述空间（不能满到演讲者被 PPT 抢话） |
| 导读副标题 | **保留**（同阅读版·论证页必有 subtitle） | 预读阶段没有讲解人在场，导读是自读的拐杖 |
| 备注 | **按现场讲写**（完整讲述逻辑·同演讲版标准） | 支撑"看完再讲"的那场会——备注是讲述脚本，不是可省项 |

**节奏 = 演讲版口径的柔化版**：**要转场**（章节骨架对自读、现场讲都是刚需），**不强制情绪页**（客户已预读过内容，现场情绪靠讲述制造，不必靠满屏 slogan 页；要用也可以，不做硬性节奏要求）。机检分流：`check_rhythm` 对预读讲解版只查转场缺失、不查情绪页；`check_reading_subtitle` 对它照查导读副标题（与阅读版同口径）。

**与两态的边界（怎么判该选它）**：
- 只被现场讲、不单独发出去自读 → **演讲版**；
- 只被自读、没有"由我们来讲"的那场会（邮件评审 / 留底 / 传阅）→ **阅读版**；
- **先发出去自读 + 之后还有一场由我们讲** → **预读讲解版**（两个条件缺一不可，只满足其一就退回对应旧态）；
- 预算/周期撑得起、且要"现场极简震撼 + 会后留底"两种极致体验的项目，照旧出**两版**（演讲版 + 阅读版·同一 storyline）——预读讲解版是"一份 deck 走完两个阶段"的折中形态，不是两版方案的替代品。

## 五、双模式新增页型（→ 范本卡库）

| 页型 | 模式 | 何时用 | 手法 |
|---|---|---|---|
| **转场页** | 演讲 | 章节切换 | 纯标题 + 编号，留白，做节奏骨架 |
| **情绪/slogan 页** | 演讲 | 态度 / 洞察 / 转折 | 大字一句 + 一张图，**无数据**，给演讲者发挥 |
| **数据 callout 页** | 演讲 | 单一关键数 | 巨大数字 + 一句解读 |
| **高潮页** | 演讲 | 情感顶点 | 全屏视觉冲击 |
| **落幕页** | 演讲 | 收尾 | 呼应开篇，留余韵 |
| **导读副标题页** | 阅读 | 几乎每页 | 标题下一行说明本页作用 |

## 六、和既有方法论的接口

- **双模式是 storyline（[03](03_场景叙事结构.md)）之上的呈现层开关**：storyline（论点 + 一组证据 + framing）不变，演讲/阅读只决定**怎么呈现**。
- 演讲版"一页一观点"呼应 [01](01_顶级deck共因与翻车雷区.md) 共因1（action title 论断标题）；情绪页是"每页 = 演讲者的汇总"的体现。
- 阅读版"导读副标题" = 01 的 so-what 显性化（每页明确告诉读者本页作用）。
- 演讲版"一组证据合围 + framing"仍成立，但证据可拆成多页递进（演讲版每页一个证据），阅读版一页合围（密度高）。
- **和 [08 deck mode](08_deck_mode叙事骨架体系.md) 正交**：07 管"长什么样"（本文档），08 管"怎么论证"——两条独立锁定的轴，任意组合都合法，不是 2×2 矩阵。
- **第三态与两态的接口（D19 FR3）**：预读讲解版的导读副标题规则**继承阅读版口径**（`check_reading_subtitle` 同查）、节奏**继承演讲版柔化口径**（`check_rhythm` 要转场不强制情绪页）、密度取中；"storyline 不变、只换呈现"与 08 正交原则对第三态同样成立。枚举落点：`storyline_state.DELIVERY_PURPOSES` 三值（含内联同步副本 `deck_rules/storyline._DELIVERY_VALUES`·值域改动两处必须同步）。

> **待巩固**：① 仅 2 份样本，多拆几份升格硬规则 ② 演讲版各页型的精确版式参数（字号/留白比/配色值）待单页高清核验 ③ 与 02 设计系统、03 节奏的交叉引用 ④ 第三态「预读讲解版」仅用户场景背书（2026-07-02 第二轮实测提出），待第三轮及以后实测的真实 deck 拆解巩固升格。
